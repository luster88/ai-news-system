### 個人で作ったMCPサーバーを公式レジストリに公開するまで — npm publish と mcp-publisher の2段階

個人開発したTypeScript製MCPサーバーをnpmと公式レジストリの両方に公開した実例記事。npm publish後にmcp-publisherでメタデータ登録する2段階手順を解説。package.json.mcpNameとserver.json.nameの完全一致、server.jsonの手動同期、description100文字制限、npxキャッシュ問題などのハマりどころを共有。レジストリ掲載で得られるのは発見性とブランドであり、ダウンロード数はbot水増しが多く実ユーザー数は1〜2桁少ないと冷静に分析している。
